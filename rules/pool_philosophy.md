# 观察池哲学与规则 v2.0

## 核心定位

观察池不是"热点收藏夹"，而是**逻辑实验场**。

```
选股逻辑假设 → 入池观察 → 验证结果 → 修正逻辑 → 复利积累
```

**目的**：通过持续验证，建立可复制的选股逻辑体系。

---

## 一、入池规则

### 1.1 入池必须满足（缺一不可）

| 条件 | 说明 | 示例 |
|------|------|------|
| **逻辑假设** | 为什么选它？预期什么？ | "AI算力需求爆发→光模块业绩兑现" |
| **验证时间** | 什么时候验证？ | "1个月后看Q2财报" |
| **成功标准** | 什么算逻辑兑现？ | "营收增速>30%，毛利率提升" |
| **失败标准** | 什么算逻辑证伪？ | "营收增速<10%，或行业需求下降" |

### 1.2 入池逻辑分类

| 逻辑类型 | 说明 | 验证周期 |
|----------|------|----------|
| **业绩驱动** | 预期财报超预期 | 季报后 |
| **行业周期** | 行业景气度上行 | 1-3个月 |
| **政策催化** | 政策利好落地 | 1-2个月 |
| **技术突破** | 新技术/新产品放量 | 2-3个月 |
| **估值修复** | 低估+催化剂 | 1-2个月 |
| **事件驱动** | 并购/重组/订单 | 事件落地后 |

## 二、观察规则

### 2.1 观察期行为

| 行为 | 说明 |
|------|------|
| **跟踪基本面** | 行业数据、公司公告、竞争对手动态 |
| **跟踪技术面** | 价格走势、成交量、资金流向 |
| **记录变化** | 任何影响逻辑假设的变化都要记录 |
| **不轻易加仓** | 观察期内不操作，只观察 |

### 2.2 观察期时长

| 逻辑类型 | 最短观察期 | 最长观察期 |
|----------|------------|------------|
| 业绩驱动 | 1个月 | 3个月 |
| 行业周期 | 2个月 | 6个月 |
| 政策催化 | 1个月 | 2个月 |
| 技术突破 | 2个月 | 4个月 |
| 估值修复 | 1个月 | 2个月 |
| 事件驱动 | 事件落地 | 1个月 |

## 三、验证规则

### 3.1 验证时间点

- **定期验证**：每周复盘时检查所有观察池股票
- **触发验证**：重大事件发生时立即验证
- **到期验证**：到达verify_date时必须验证

### 3.2 验证结果分类

| 结果 | 定义 | 处理 |
|------|------|------|
| ✅ **逻辑兑现** | 成功标准达成 | 升级到A/B池，记录成功模式 |
| ⏳ **逻辑进行中** | 还没到验证时间 | 继续观察 |
| ⚠️ **逻辑偏移** | 部分达成，部分偏离 | 分析原因，调整预期 |
| ❌ **逻辑证伪** | 失败标准达成 | 退池，记录失败原因 |
| 🔄 **逻辑修正** | 新信息改变假设 | 更新逻辑，重置验证时间 |

### 3.3 验证模板

```json
{

  "lesson": "AI算力需求确实带动光模块放量，逻辑成立",
  "action": "升级B池，继续持有"
}
```

## 四、复盘规则

### 4.1 周复盘（每周五收盘后）

1. **检查观察池**：哪些股票到了验证时间？
2. **验证逻辑**：成功/失败/偏移？
3. **记录教训**：为什么对？为什么错？
4. **更新规则**：是否需要调整选股逻辑？

### 4.2 月复盘（每月最后一个交易日）

1. **统计胜率**：本月验证了多少？成功多少？
2. **分析模式**：哪些逻辑类型胜率高？
3. **优化体系**：调整入池标准、验证周期
4. **清理池子**：长期未验证的股票强制验证或退池

### 4.3 复盘模板

```markdown
## 2026年6月观察池复盘

### 本月验证
- 入池：12只
- 验证：5只
- 成功：3只（60%）
- 失败：2只（40%）

### 成功模式
- 业绩驱动：2/2 成功（100%）
- 行业周期：1/2 成功（50%）
- 事件驱动：0/1 成功（0%）

### 失败教训
- 002361 神剑股份：逻辑是"军工订单放量"，但实际是游资炒作，无基本面支撑
- 教训：事件驱动类需验证"是否有真实订单"，不能只看涨停

### 规则调整
- 事件驱动类入池需增加"订单/合同验证"条件
- 涨停股入池需等待3天观察是否有持续性
```

## 五、池子分类（重新定义）

| 池子 | 定义 | 入池条件 | 止损线 |
|------|------|----------|--------|
| **A池** | 逻辑已验证的核心持仓 | 逻辑兑现+趋势良好 | -10% |
| **B池** | 逻辑进行中的观察池 | 有明确逻辑假设 | -5% |
| **C池** | 新逻辑的试验池 | 新入池，待观察 | -3% |
| **退池** | 逻辑证伪或止损触发 | 自动/手动退池 | - |

### 池子流动规则

```
新入池 → C池（观察期）
  ↓ 逻辑进行中
B池（跟踪期）
  ↓ 逻辑兑现
A池（核心持仓）
  ↓ 逻辑证伪/止损
退池
```

## 六、与OCIFQ的关系

OCIFQ是**选股工具**，观察池是**验证工具**。

```
OCIFQ选股 → 逻辑假设 → 观察池验证 → 修正OCIFQ权重
```

**闭环**：
1. 用OCIFQ选出符合标准的股票
2. 形成逻辑假设入池观察
3. 验证结果反馈给OCIFQ
4. 调整OCIFQ各维度权重

---

## 七、纪律

1. **无逻辑不入池**：没有明确假设的股票不入池
2. **有验证必记录**：每次验证都要写结果和教训
3. **止损是铁律**：跌破止损线无条件退池，不找借口
4. **复盘是必须**：每周必须复盘，不能跳过
5. **规则可进化**：根据验证结果持续优化规则

---

**核心思想**：观察池不是为了"抓住涨停"，而是为了"验证逻辑"。胜率比收益率更重要。
