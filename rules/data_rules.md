# data_rules.md — 数据规则

## 数据铁律

禁止编造任何市场数据。查不到标"数据缺失"。

搜索必须用完整日期。

如果某数据缺失，不参与计算。

情绪温度缺2项以上 → 输出"情绪温度暂估：数据不足，不精确"并列出缺失项。

禁止出现"数据缺失，但情绪温度=精确数值"。

## 连板规则

连板数据必须严格区分：
- "18连板" = 连续18个交易日涨停
- "18天9板" = 18个交易日内9次涨停 ≠ 18连板
- 炸板 ≠ 成功涨停

输出格式：代码 | 名称 | 口径：几天几板/几连板 | 是否炸板 | 来源状态

不确定时写：【连板数据存疑，暂不用于情绪温度计算】

## 事实校验优先级

1. 日期
2. 收盘/午盘口径
3. 数据来源
4. 多源冲突检查

禁止混用午盘和收盘数据。

## 纠错格式

【纠错】原写法 → 修正后 → 原因 → 影响

## 数据源降级策略

```
Level 1: 东方财富API
  ↓ 失败
Level 2: 新浪财经API
  ↓ 失败
Level 3: 上交所/深交所官网
  ↓ 失败
Level 4: 标注"数据缺失"
```

**API偶发全0/超时处理**：
- `stock_zh_a_spot()` 若 up=0 且 down=0 且 flat=total → 判定为盘前快照全0
- 此情况改查 T-1 日期涨跌停数据，market 标为"T-1收盘数据"
- 涨跌停池API若返回空DataFrame → retry 3次后标注"涨跌停数据缺失"
- 炸板池API若返回None → 炸板率用涨停池数据自己算: `炸板/(涨停+炸板)`
- 所有API调用外层必须有 try/except，单接口崩溃不中断后续采集

## 日期口径铁律

- `stock_zh_a_spot()` 返回**实时快照**，盘前(9:30前)可能全0，不可作为T-1数据
- 涨跌停池/强势股池接口用 **T-1交易日** 日期（盘前时段当日还没数据）
- 指数日线接口用 **T-1交易日** 日期过滤
- 采集时间 >15:30 → 用当日日期；采集时间 <9:30 → 用T-1日期
- 若 date.today() 是交易日但 market 数据全0 → 改查 T-1 日期重试

## 涨停阈值板块区分（2026-05-14新增）

不同板块涨跌停幅度不同，计算涨停/跌停数时必须区分：

| 板块 | 涨跌停幅度 | 代码特征 |
|------|-----------|----------|
| 主板（沪深） | ±10% | 600xxx/601xxx/603xxx/000xxx/002xxx/003xxx |
| 创业板 | ±20% | 300xxx |
| 科创板 | ±20% | 688xxx |
| 北交所 | ±30% | 8xxxxx |
| ST股票 | ±5% | 名称含ST |

情绪温度计算中，涨停判定应使用对应板块的阈值，而非统一使用9.9%。

## RAG幻觉检测规则（2026-05-14新增）

**背景**：基准测试优异的RAG Agent生产环境幻觉率高达30%（来源：AIHOT日报引用BestBlogs早报）。

### 幻觉检测清单

在生成研报/分析时，必须执行以下检查：

1. **数据源验证**
   - 所有数字（价格、涨跌幅、成交量）必须来自缓存数据或API返回
   - 禁止凭记忆/推测填写数据
   - 数据必须标注来源（脚本名+采集时间）

2. **时间一致性**
   - 引用的新闻/事件必须有明确日期
   - 禁止将历史事件误认为当前事件
   - 检查"今天/昨日/本周"等时间词的准确性

3. **逻辑一致性**
   - 结论必须与提供的数据一致
   - 禁止数据说A，结论说B的情况
   - 多个数据源冲突时必须标注

4. **引用规范**
   - 引用研报/新闻必须有来源URL或文件路径
   - 禁止编造不存在的研报/分析师观点
   - 引用数字时必须可追溯到原始数据

### 幻觉报告格式

发现幻觉时，使用以下格式记录：

```
【幻觉报告】
时间：YYYY-MM-DD HH:MM
类型：数据错误/时间错误/逻辑错误/引用错误
原文：<包含幻觉的内容>
修正：<正确内容>
原因：<幻觉产生的原因>
影响：<对报告结论的影响程度>
```

### 幻觉率监控

- 每周统计幻觉报告数量
- 目标：幻觉率 < 5%
- 超过10%需复盘模型prompt和数据管线

## 数据与方法能力

我可以处理：表格整理、指标计算、数据清洗思路、统计分析、模型选择、图表建议、结果解释、方法适配性判断、研究设计、指标体系搭建、数据口径审查。

### 数据任务原则

涉及数据时，必须说明：数据来源、数据口径、样本范围、时间范围、缺失值处理、计算方法、指标含义、结果限制。

### 方法选择原则

方法必须服务问题。不要为了显得高级而使用复杂模型。

优先判断：
1. 问题是什么？
2. 数据够不够？
3. 变量是否可观测？
4. 样本量是否支持方法？
5. 结果是否可解释？
6. 是否存在因果误读？
7. 是否存在样本偏差？
