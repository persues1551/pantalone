## 买入规则（2026-05-14 建立）

**核心原则**：买入不是拍脑袋，而是量化评分+分批建仓+大盘过滤的系统决策。

### 一、买入信号评分系统（满分100分）

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| 板块强度 | 25分 | 当日板块涨幅>1%=20分，>2%=25分，<0%=0分 |
| 技术位置 | 25分 | RSI 40-60=25分，60-70=20分，>75=5分（超买扣分） |
| 量价配合 | 20分 | 放量突破=20分，缩量上涨=10分，放量下跌=0分 |
| 消息催化 | 15分 | 重大利好(+4以上)=15分，一般利好=10分，无消息=5分 |
| 资金流向 | 15分 | 主力净流入>5000万=15分，>0=10分，净流出=0分 |

**买入门槛**：总分≥60分才执行买入。

### 二、仓位计算规则

| 评分 | 仓位比例 | 金额范围（20万本金） |
|------|----------|---------------------|
| 60-69分 | 10% | 2万 |
| 70-79分 | 15% | 3万 |
| 80-89分 | 20% | 4万 |
| 90+分 | 25%（单票上限） | 5万 |

**高价股例外**：如果1手（100股）金额超过25%仓位上限，允许买入1手（仅1手，不超额）。例：新易盛563.70元/股，1手=56,370元，超过5万上限，但允许买入1手。

### 三、分批建仓规则

| 条件 | 操作 |
|------|------|
| 评分≥60且大盘情绪≥5/10 | 第一批买入50%计划仓位 |
| 次日确认（不跌>2%） | 第二批买入剩余50% |
| 次日跌>2% | 取消第二批，观察 |
| 大盘情绪<5/10 | 不建仓，仅观察 |

### 四、大盘环境过滤器

| 大盘状态 | 判断标准 | 操作 |
|----------|----------|------|
| 🟢 主升浪 | 创业板连涨2日+涨停>80 | 积极建仓，暴露可达70% |
| 🟡 震荡 | 指数横盘+涨停50-80 | 正常建仓，暴露≤50% |
| 🔴 回调 | 指数连跌2日+涨停<50 | 不建仓，等待企稳 |
| ⚫ 恐慌 | 指数跌>2%+跌停>20 | 不建仓，考虑减仓 |

### 五、标的选择优先级

同一天多个标的满足买入条件时：
1. 评分最高的优先
2. 评分相同→板块强度更高的优先
3. 板块相同→市值更大的优先（流动性更好）
4. 同一方向最多执行2只

### 六、禁止买入条件（硬约束）

| 条件 | 理由 |
|------|------|
| RSI>75 | 超买区，追高风险大 |
| 连续3日缩量上涨 | 量价背离，随时回调 |
| 高开5%以上 | 追高成本太高 |
| 涨停板打开 | 主力出货信号 |
| ST/退市风险 | 系统性风险 |
| 数据缺失>3项 | 无法判断 |

## 三层清仓逻辑（2026-05-14 建立）

**核心原则**：清仓不是单一触发器，而是三层判断体系。每日收盘后按顺序检查，任一条件触发即执行。

### 第一层：趋势跟踪（均线系统）

| 条件 | 操作 | 理由 |
|------|------|------|
| 股价站稳5日均线上方 | 持有不动 | 短期趋势完好 |
| 跌破5日均线 | 减半仓 | 短期趋势转弱 |
| 跌破10日均线 | 清仓 | 中期趋势转弱 |
| 跌破5日后次日收回 | 不减仓（假破） | 避免洗盘被震出 |

**5日均线计算**：取最近5个交易日收盘价的算术平均值。

### 第二层：分批止盈（锁定利润）

| 浮盈 | 操作 | 理由 |
|------|------|------|
| +12% | 卖出1/3 | 避免过早止盈，给趋势空间 |
| +20% | 再卖出1/3 | 大部分利润已锁定 |
| +30% | 剩余1/3跟踪止盈 | 让利润奔跑 |
| 回撤至最高点-5% | 卖出剩余 | 趋势结束信号 |

**注意**：分批止盈按股数取整至100股。若持仓仅100股，+12%时卖出全部（无法分1/3）。

### 第三层：时间窗口（匹配板块轮动）

| 持仓天数 | 操作 | 理由 |
|----------|------|------|
| 1-3天 | 正常持有 | 板块轮动周期内 |
| 3-5天 | 开始关注止盈信号 | 轮动窗口即将关闭 |
| 5-7天 | 无条件减半仓 | 超过轮动周期 |
| 10天+ | 无条件清仓 | 避免长期套牢 |

**板块轮动周期依据**：5/6-5/14回放数据显示，A股板块轮动周期约2-3天，主线个股最大涨幅通常在3-5天内完成。

### 综合清仓判断流程（每日收盘后执行）

```
1. 股价是否跌破5日均线？→ 减半仓
2. 股价是否跌破10日均线？→ 清仓
3. 浮盈是否达到+8%/+15%/+25%？→ 分批止盈
4. 持仓是否超过5天？→ 关注止盈信号
5. 持仓是否超过10天？→ 无条件清仓
6. 板块资金是否连续2日流出？→ 减仓
以上任一条件触发即执行，不犹豫。
```

### 历史验证

| 股票 | 买入 | 旧规则结果 | 新规则预期 | 改进 |
|------|------|-----------|-----------|------|

## Token 优化（v1.14）

> **教训（2026-05-12 审计）**：实测全天消耗 1,162 万 token（常规输入），加上推理 token 后平台账单约 **5,000 万/天**。输出仅占 <1%——输出压缩是伪需求。真正的大头：①上下文累积（85%）②推理 token（reasoning_effort: medium 产生 3-8x 隐性消耗）③工具定义重发（10%）。

> **主人决策（2026-05-12）**：**质量优先于成本。** 投资分析质量不得为省 token 而妥协。优化方向仅限于**不影响分析质量的基础设施层**：工具裁剪、会话管理、缓存利用。

### Cron Job 工具集限制（必须）

**所有 Amadeus cron job 必须设置 `enabled_toolsets`**，只保留实际用到的工具：

```yaml
enabled_toolsets: ["terminal", "web", "file"]
```

效果：工具定义从 29 个（~11,400 tok/轮）→ 4-6 个（~1,500 tok/轮），每 session 省 ~10,000 token。

## 会话管理纪律

- **任何会话超过 15 轮（user turns）→ 主动 `/new` 重置上下文**
- 44 轮长会话实测消耗 308 万 token，拆成 3 个 15 轮仅需 ~150 万 token
- 完成一个独立任务后（如修复脚本、调试 cron），立即 `/new` 开始下一个任务
- cron 推送会话天然只有 1 轮，无需处理

## 输出不优化原则

- 输出 token 占比 <1%，压缩输出是 ROI 最低的优化方向
- 不要为了省 token 而降低分析质量、删除风险声明、压缩数据核验
- 如果未来输出显著增长（如 deep research 模式），再评估

## DeepSeek Context Caching（已自动启用）

DeepSeek KV Cache **默认开启**，无代码修改：缓存命中仅收原价 1/10。但要求**精确前缀匹配**——长会话中前缀不断膨胀，缓存命中率骤降。

## 推理 Token（沉默杀手）

`reasoning_effort: medium` 使模型每输出 1 token 前做 3-8x 内部推理。推理 token 不在 session JSON 中，只出现在平台账单。

支持 effort 值：`none`/`minimal`/`low`/`medium`/`high`/`xhigh`。

简易任务（盘前早报、系统检查）可用 `low` 省 40%，但目前不支持 per-job 设置。
