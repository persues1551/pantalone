# checklist.md — 报告生成自检清单（降级版 v2.0）

**注意**：本文件已从"Subagent防假规则"降级为"报告生成自检清单"。
不存在独立的Subagent调用，所有分析由单一LLM完成。

## 报告自检清单（每次生成报告前必须检查）

### 数据完整性
- [ ] 指数数据是否来自缓存/API（非编造）？
- [ ] 涨跌停数据是否区分板块阈值（主板10%/科创创业20%/北交30%）？
- [ ] 成交额是否有变化率对比（vs前N日均值）？
- [ ] 技术指标（RSI/MACD/MA）是否从indicators JSON读取？

### 模拟盘
- [ ] 是否调用了 `amadeus_sim_integrate.py status`？
- [ ] 持仓浮盈浮亏是否使用实时价格（非买入价）？
- [ ] 收盘报告是否调用了 `daily_update`？
- [ ] 买入计划是否通过评分系统（≥60分）？
- [ ] 单票金额是否≤min(25%总仓位, 5万)？
- [ ] 总暴露是否≤大盘过滤器允许的最大暴露？

### 风控
- [ ] 是否检查了三层清仓条件（均线/止盈/时间窗口）？
- [ ] 是否检查了禁止买入条件（RSI>75/缩量/高开5%+）？
- [ ] 大盘环境是否通过过滤器判定？

### 报告格式
- [ ] 是否包含模拟盘部分？
- [ ] 是否标注数据来源？
- [ ] 是否包含风险声明？
- [ ] 是否去除了Subagent分工标签？

### 排雷
- [ ] 新买入标的是否通过排雷扫描？
- [ ] 数据缺失项是否标注？
