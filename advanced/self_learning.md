# self_learning.md — 自学习机制

## AIHOT日报自我迭代分析（每日必做）

每天获取AIHOT AI日报后，**自动执行**以下分析流程：

### 1. 扫描

逐条筛选日报内容，判断是否与Amadeus/Hermes系统能力迭代相关。

### 2. 相关维度

| 维度 | 说明 |
|------|------|
| AI Agent框架/工具/方法论 | 状态机、上下文管理、多Agent协作等 |
| 数据获取/分析新工具 | 新API、新数据源、爬虫技术等 |
| 自动化/工作流改进 | 任务调度、并行化、容错机制等 |
| 金融/投资相关AI应用 | 量化、预测、情绪分析等 |
| 代码生成/调试/优化新能力 | prompt优化、token节省、模型路由等 |
| 医学科研新工具 | PubMed增强、文献分析、临床数据等 |

### 3. 评估

对每条相关发现评估三个维度：
- (a) 能否帮助系统改进（高/中/低）
- (b) 实现难度（简单≤1天/中等2-3天/困难>3天）
- (c) 具体实现方案（一句话）

### 4. 行动

| 条件 | 动作 |
|------|------|
| 简单(≤1天) + 高价值 | **当天直接实现** |
| 中等/困难 | 写入TODO队列，标注优先级(P0/P1/P2) |
| 发现新工具/新API | 测试可用性，可用则接入 |

### 5. 输出格式

```
【AIHOT日报迭代扫描 · YYYY-MM-DD】
扫描N条，发现M条相关：
🔴 可立即实施：X条 → [已完成/已加入TODO]
🟡 值得关注：Y条 → [观察/调研]
🟢 了解即可：Z条
```

### 6. 铁律

不得因为"觉得没必要"而跳过扫描。即使当天无相关发现，也要输出"扫描完成，无相关发现"。

## 系统迭代原则

复盘或日常使用中发现规则缺陷、脚本bug、数据采集漏洞 → **直接修复，不提问不请示**。

包括但不限于：
- 脚本逻辑修复
- skill规则更新
- 数据采集流程改进
- 情绪温度公式调整
- 观察池规则变更
- 模板格式修正

修复完成后简明汇报变更内容，不要铺垫"建议"或"可以考虑"。

## 迭代记录

每次迭代后，记录到 `logs/update_log.md`：

```markdown
## YYYY-MM-DD - 迭代标题

**来源**：AIHOT日报 / 复盘发现 / 主人反馈 / 系统故障
**类型**：规则变更 / 脚本修复 / 配置变更 / 新增功能
**变更内容**：具体变更
**验证**：如何验证变更正确
```

## 迭代优先级

| 优先级 | 条件 | 处理方式 |
|--------|------|----------|
| P0 | 影响核心功能/数据准确性 | 当天立即修复 |
| P1 | 影响效率/体验 | 1-3天内处理 |
| P2 | 优化/增强类 | 纳入TODO队列 |
| P3 | 长期规划 | 记录但不立即执行 |
