# 持仓诊断工作流

当主人说"研究一下我的持仓"或提供持仓截图时，按以下流程执行。

## 触发条件

- 主人发送持仓截图
- 主人说"分析持仓"、"看看我的持仓"、"诊断持仓"
- 主人提供股票代码列表要求组合分析

## 四步诊断流程

### 第一步：数据采集（并行）

1. **实时行情**：腾讯API批量获取所有持仓的现价、涨跌、PE、换手率
2. **技术指标**：新浪K线API获取历史数据，计算MA5/10/20/50/120
3. **ETF成分**：如有ETF持仓，搜索前十大持仓了解实际暴露

```bash
# 腾讯实时行情
curl -s "http://qt.gtimg.cn/q={codes}" | iconv -f gbk -t utf-8

# 新浪K线（计算MA）
curl -s "https://quotes.sina.cn/cn/api/jsonp_v2.php/var/CN_MarketDataService.getKLineData?symbol={code}&scale=240&ma=no&datalen=130"
```

### 第二步：逐只分析

对每只持仓输出：
| 指标 | 说明 |
|------|------|
| 现价/成本/盈亏 | 基础盈亏 |
| PE/PB | 估值水平 |
| 距MA50/MA120 | 技术位置（左侧/右侧） |
| 近5日/近1月涨跌 | 短期动量 |
| 换手率 | 活跃度（选择MA参考线） |
| OCIFQ评分（个股） | 基本面质量 |

### 第三步：组合层面分析

**必须检查的5个维度**：

1. **行业集中度**：所有持仓是否集中在同一赛道？
   - 警戒线：单一赛道>60% → 高度集中
   - 铁律：单一赛道>80% → 必须警告

2. **ETF重叠度**：多个ETF是否持有相同标的？
   - 检查方法：对比ETF前十大持仓
   - 常见重叠：科创50ETF ∩ 科创芯片ETF（科创板芯片是科创50核心权重）
   - 常见重叠：半导体设备ETF ∩ 科创50ETF（北方华创、中微公司）

3. **仓位分布**：最大单票占比是否过大？
   - 警戒线：单票>30% → 仓位过重
   - 建议：单票<25%

4. **攻防配比**：进攻型（科技/成长）vs 防御型（消费/高股息/资源）比例
   - 100%进攻 → 无防御，全军覆没风险
   - 建议：至少20%防御仓位

5. **技术面风险**：有多少持仓处于远高于均线（追涨风险）？
   - 距MA120>+15% → 追涨风险大
   - 近5日全部-7% → 板块获利回吐

### 第四步：调仓建议

基于以上分析，给出具体建议：

| 情形 | 建议 |
|------|------|
| 亏损股+高PE+技术面弱 | 考虑止损/减仓 |
| 盈利丰厚+远高于均线 | 考虑止盈/减仓 |
| ETF高度重叠 | 合并到流动性更好的那只 |
| 仓位过重 | 减仓到25%以下 |
| 缺少防御仓位 | 加入消费/高股息/资源品 |

## 输出格式

```
## 持仓诊断

### 持仓明细表（含盈亏、技术面、估值）

### 逐只诊断（每只2-3行关键判断）

### 组合核心问题（按严重程度排序）

### 调仓建议（具体到股数/金额）
```

## Pitfalls

1. **ETF没有PE**：ETF的PE字段通常为空，不需要强行计算
2. **ETF成本可能不准**：用户截图中的成本可能是复权后价格，计算盈亏时注意
3. **科创50和科创芯片高度重叠**：这是最常见的ETF重叠组合，必须提醒
4. **光启技术等军工股PE极高是常态**：不能简单用PE判断，需结合行业特性
5. **换手率低的个股用MA120**：不要用MA50，否则参考线太短看不到长期趋势
6. **不要替用户做最终决定**：给出分析和建议，但决策权在用户
