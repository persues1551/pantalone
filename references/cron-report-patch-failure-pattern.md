# Cron报告patch失败模式（2026-05-25复盘）

**问题**：收盘复盘cron job生成报告后，尝试用patch修改章节名（"池别汇总"），但报告中实际章节名是"观察池自动管理"，导致hunk not found错误。

**根因**：cron agent生成报告时章节名不固定，后续patch用的old_string与实际生成内容不匹配。

**修复原则**：
1. **一次性写入，禁止patch** — prompt中明确"一次性写入完整报告，不要后续patch修改"
2. **强制章节名** — prompt中用加粗标注必须使用的章节名，禁止用其他名称
3. **禁止模糊替代** — 如禁止"池别汇总"，必须用"观察池自动管理"

**示例prompt片段**：
```
- 🎲 **观察池表现**（涨跌榜/异动回顾）——注意章节名必须是"观察池表现"
- 📝 **观察池自动管理**（退池/入池/当前池内统计）——禁止使用"池别汇总"
- **一次性写入完整报告，禁止后续patch修改**
```

**适用范围**：所有cron报告类任务（早报/午盘/收盘复盘/周报），以及任何需要生成结构化文档的cron job。

**经验教训**：
- Cron agent的输出不确定性比交互式session更高（无用户反馈修正机会）
- 结构化文档生成应尽量在prompt中固定格式，而非依赖agent"记住"模板
- Patch操作在cron场景下风险更高，应尽量避免
