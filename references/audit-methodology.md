# Amadeus 12维度审计框架

> 2026-05-14 首次全面审计，评分4.4/10→修复后6.5/10→v3.1预计7.5/10

## 审计维度

| 维度 | 检查内容 | 评分标准 |
|------|----------|----------|
| 1. 数据采集 | 数据源稳定性、降级逻辑、一键脚本正确性 | 有脚本+有降级=7+ |
| 2. 财报分析 | 估值模型、排雷检查、财报采集完整性 | 有脚本=6+，无脚本=4以下 |
| 3. 情绪分析 | 涨跌停数据源、情绪温度公式、验证准确率 | 公式正确+数据源可靠=7+ |
| 4. 题材分析 | 板块资金流、三项检查、轮动周期 | 有资金流数据=6+，缺失=4以下 |
| 5. 技术触发 | RSI/MACD/MA是否写入JSON、触发条件脚本化 | 入JSON+有脚本=7+ |
| 6. 模拟盘 | 数据库是否被cron写入、实时价格、评分系统 | 有写入+有评分=7+，未写入=2 |
| 7. 风控 | Risk Agent否决权、大盘过滤器、禁止买入检查 | 有脚本=6+，未实现=4以下 |
| 8. 组合管理 | 仓位分配、暴露控制、单票上限 | 有自动检查=6+，依赖模拟盘=3 |
| 9. 评分复盘 | 预测维度验证、准确率统计、改进闭环 | 5/5可验证=7+，3/5=5 |
| 10. Subagent | 权限边界、执行纪律、防假规则 | 有实现=6+，全空壳=3 |
| 11. 模型路由 | L0-L4分级、路由冲突、视觉路由 | 无冲突=7+，有冲突=5 |
| 12. 自动化 | cron执行、API健康检查、中断恢复 | 有备份+有降级=7+ |

## 两类核心问题

### Execution Gap（规则写了但没执行）
- 模拟盘规则迭代3版，数据库0条记录
- Subagent架构11个Agent全是markdown文件
- 排雷检查有清单但无脚本实现
- **诊断方法**：grep规则文件中的"必须"关键词，检查对应脚本是否实际调用

### Data Pipeline Gap（数据采集有但使用断裂）
- RSI/MACD只print不写JSON
- 板块资金流完全不采
- 排雷检查无脚本
- **诊断方法**：检查数据采集脚本的输出是否被分析脚本消费

## 修复优先级

| 优先级 | 标准 | 示例 |
|--------|------|------|
| P0 | 核心链路断裂/硬bug | 模拟盘未写入/脚本调用废弃文件 |
| P1 | 数据缺失/规则冲突 | 板块资金流缺失/router冲突 |
| P2 | 优化项/文档清理 | 样本不足/缓存清理 |

## 防复发机制

每个修复必须附带防复发机制：
- 脚本级：自动检查/拦截/降级
- 规则级：写入rules文件+同步所有相关文件
- cron级：嵌入执行步骤
- 文档级：更新SKILL.md pitfalls
