# 观察池自动扫描机制

## 概述

自动扫描机制定期扫描全市场股票，自动入池符合条件的标的，解决"观察池只被动管理"的问题。

---

## 一、扫描类型

### 1.1 ML信号扫描

- **时间**：每天16:00（收盘后）
- **范围**：观察池活跃股票
- **条件**：ML评分>60，信号为"买入"
- **入池**：C池，验证期30天

**逻辑假设**：
```json
{
  "type": "业绩驱动",
  "hypothesis": "ML模型评分70，模型看好短期趋势",
  "catalyst": "ML模型预测",
  "success_criteria": "1个月内涨幅>10%",
  "failure_criteria": "1个月内跌幅>5%",
  "confidence": 0.7,
  "verify_days": 30
}
```

### 1.2 行业轮动扫描

- **时间**：每天16:30（收盘后）
- **范围**：资金流入前3行业
- **条件**：行业龙头，市值前3
- **入池**：C池，验证期60天

**逻辑假设**：
```json
{
  "type": "行业周期",
  "hypothesis": "半导体行业龙头，行业涨幅3.5%",
  "catalyst": "半导体行业资金流入",
  "success_criteria": "行业持续景气，公司业绩增长",
  "failure_criteria": "行业转冷，或公司业绩不及预期",
  "confidence": 0.6,
  "verify_days": 60
}
```

### 1.3 新闻热点扫描

- **时间**：每天17:00（收盘后）
- **范围**：当日新闻热点
- **条件**：有股票代码提及
- **入池**：C池，验证期14天

**逻辑假设**：
```json
{
  "type": "事件驱动",
  "hypothesis": "新闻热点催化: 新易盛1.6T光模块进入大规模交付期",
  "catalyst": "新闻事件",
  "success_criteria": "事件持续发酵，股价上涨",
  "failure_criteria": "事件消退，或被证伪",
  "confidence": 0.5,
  "verify_days": 14
}
```

## 二、扫描脚本

### 2.1 脚本位置

`~/.hermes/scripts/amadeus/pool_auto_scanner.py`

### 2.2 使用命令

```bash
# 单类型扫描
python3 pool_auto_scanner.py --ml-only --max 5
python3 pool_auto_scanner.py --industry-only --max 5
python3 pool_auto_scanner.py --news-only --max 5

# 全量扫描
python3 pool_auto_scanner.py --max 10

# 指定类型
python3 pool_auto_scanner.py --types ml industry news --max 10
```

### 2.3 参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| --types | 扫描类型列表 | ml, industry, news |
| --max | 最大入池数量 | 10 |
| --ml-only | 只扫描ML | false |
| --industry-only | 只扫描行业 | false |
| --news-only | 只扫描新闻 | false |

## 三、定时任务配置

### 3.1 任务列表

| 任务 | 时间 | 频率 | 最大入池 | Job ID |
|------|------|------|----------|--------|
| ML扫描 | 16:00 | 每个交易日 | 5只 | 4528827678fc |

| 新闻扫描 | 17:00 | 每个交易日 | 5只 | 8b4c6c5326f6 |

### 3.2 任务间隔

- ML扫描 → 行业扫描：30分钟间隔
- 行业扫描 → 新闻扫描：30分钟间隔
- 避免API限流

### 3.3 管理命令

```bash
# 查看任务
hermes cron list

# 暂停任务
hermes cron pause JOB_ID

# 恢复任务
hermes cron resume JOB_ID

# 手动触发
hermes cron run JOB_ID
```

## 四、入池流程

### 4.1 流程图

```
扫描发现候选
    ↓
过滤（排除已在池中、ST股）
    ↓
去重（按代码）
    ↓
生成逻辑假设
    ↓
入池（默认C池）
    ↓
记录扫描日志
```

### 4.2 过滤规则

1. **排除已在池中**：检查session_context.json
2. **排除ST股**：检查股票名称
3. **去重**：同一股票不重复入池

### 4.3 逻辑假设生成

根据扫描来源自动生成：
- ML信号：业绩驱动，验证期30天
- 行业龙头：行业周期，验证期60天
- 新闻热点：事件驱动，验证期14天

## 六、监控与告警

### 6.1 扫描失败告警

如果扫描失败，日志会记录错误信息：
```json
{
  "time": "2026-06-08T16:00:00",
  "types": ["ml"],
  "scanned": 0,
  "filtered": 0,
  "added": 0,
  "error": "ML预测超时"
}
```

### 6.2 入池数量告警

- 单次入池>10只：警告
- 单次入池>20只：暂停扫描，人工审核

### 6.3 重复入池告警

如果同一股票在7天内被重复扫描入池，记录告警：
```json
{
  "code": "300502",
  "name": "新易盛",
  "first_scan": "2026-06-01",
  "second_scan": "2026-06-08",
  "reason": "7天内重复入池"
}
```

## 七、优化建议

### 7.1 扫描条件优化

- **ML信号**：根据验证结果调整阈值（当前>60）
- **行业轮动**：根据行业持续性调整扫描行业数（当前前3）
- **新闻热点**：根据新闻质量过滤（当前取前10条）

### 7.2 入池策略优化

- **分批入池**：不要一次入池太多，分散风险
- **行业分散**：同行业不重复入池
- **时间分散**：不同时间入池，避免追高

### 7.3 验证机制优化

- **自动验证**：到期自动提醒验证
- **批量验证**：每周五批量验证
- **验证报告**：生成验证报告，记录教训

## 八、常见问题

### Q1: 扫描没有发现候选？

A: 可能原因：
- ML模型看空所有股票（当前市场环境）
- 行业资金流出（无热门行业）
- 新闻缓存过期（需要更新新闻）

### Q2: 入池后股价下跌？

A: 处理方式：
- 检查止损线（C池-3%）
- 如果触发止损，无条件退池
- 记录失败原因，优化扫描条件

### Q3: 扫描超时？

A: 解决方案：
- 增加超时时间（当前120秒）
- 减少扫描数量（当前--max 5）
- 分批扫描（先ML，再行业，再新闻）

### Q4: 重复入池？

A: 处理方式：
- 检查过滤逻辑
- 手动退池重复股票
- 优化去重算法
