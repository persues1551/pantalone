# 每日预测-复盘-模拟盘-自我迭代系统设计

## 系统目标

从2026年5月1日起建立完整的预测-复盘-模拟盘-自我迭代闭环系统。

## 核心原则

1. **禁止未来函数**：盘前决策只能使用盘前可获取的信息
2. **禁止编造数据**：无数据源直接标注"数据缺失"
3. **禁止盘后倒推**：不能用已知结果反推逻辑
4. **禁止真实交易**：仅模拟盘，不涉及真实资金

## 数据层级

| 数据类型 | 获取时机 | 可靠性 |
|----------|----------|--------|
| 隔夜外围 | 盘前08:00 | 高 |
| 新闻扫描 | 盘前08:30 | 中 |
| 技术指标 | 盘前09:00 | 高 |
| 指数行情 | 盘中实时 | 高 |
| 板块资金流 | 盘后 | 高 |
| 涨跌停数据 | 盘后 | 高 |

## 预测维度（5项固定）

1. **指数方向**：上证区间+偏强/偏弱/震荡
2. **资金主线**：预期净流入TOP3板块
3. **最强方向**：预期领涨板块/个股
4. **最弱方向**：预期领跌板块
5. **风险点**：触发降仓的条件

## 预测验证规则

- 每项必须有明确的真/假验证标准
- 禁止模糊表达（"可能涨""或许跌"）
- 收盘后逐项验证，计算准确率
- 连续3次某维度错误 → 标记该判断逻辑待调整

## 模拟盘规则

- 初始资金：200,000元
- 单票仓位：10,000-25,000元
- 总风险暴露：≤60%
- 买入股数：100股整数倍
- 成交价：收盘价（保守原则）
- 止损：B池-5%，A池-8%

## 每日流程

```
08:30  盘前早报 → 预测5项+模拟盘计划
09:00  盘前容错备份
09:30  新闻扫描
12:00  午间复盘 → 上午验证+下午计划
12:30  午间容错备份
15:30  收盘复盘 → 预测验证+模拟盘结算+成功点/失败点/明日优化
16:00  收盘容错备份+观察池管理
20:00  每日搁置任务提醒
```

## 历史回放规则

1. 早盘预案只能使用当日9:30前可获取的信息
2. 预测必须在盘前发布，收盘后验证
3. 模拟盘成交价取收盘价（保守原则）
4. 禁止使用"已知结果倒推逻辑"

## 回放数据源

| 5/11-5/12 | 本地缓存 | 指数/技术指标 |
| 5/13 | 本地缓存 | 完整数据+复盘报告 |
| 5/14 | 实时数据 | 早报+实时行情 |

## 自我迭代机制

1. 每次学习/解决问题后问自己：
   - 这个经验值得记忆吗？
   - 能写成skill吗？
   - 能改进模板/规则吗？
2. 不依赖主人反复提醒，主动总结教训
3. 发现规则缺陷直接修复，不问主人
4. 预测准确率每周汇总，识别系统性偏差
