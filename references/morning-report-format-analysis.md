# 参考文档分析 — A股早盘预案与复盘模板

**来源**：主人提供的两份参考文档（2026-05-14）
**用途**：学习改进 Amadeus 早报/复盘格式

## 早盘预案（20260511）结构分析

### 结构
1. 大局观（指数/情绪/题材/消息）
2. 具体机会解析（4个方向）
3. 趋势票或短线交易方法
4. 总结
5. 预案自评
6. 10只观察股票池
7. 20万元模拟账户交易计划

### 关键特点
- **情绪评分**：量化为 x/10，列出加分项/扣分项
- **题材三项检查**：盘面强度、消息催化、成交承接
- **主攻方向**：≤2个，明确排除"暂不作为主攻方向"
- **操作方法**：短线接力法 vs 趋势低吸法，各配执行条件
- **模拟盘纪律**：单票1-2.5万，总暴露≤60%，100股整数倍
- **资料来源**：列出所有数据来源

### 语言风格
- 简洁直接，不标注"XX Agent输出"
- 判断格式：「指数处在xxx阶段。今日重点不是xxx，而是xxx。」
- 每个方向配具体股票和操作方法

## 每日复盘（20260511）结构分析

### 结构
1. 今日市场真实表现
2. 板块资金流入流出分析
3. 早盘预案验证
4. 10只观察股票池T+0验证
5. 滚动台账与准确率
6. 20万元模拟账户台账
7. 成功点、失败点与明日优化
8. 资料来源与口径说明

### 关键特点
- **预案验证**：综合评分 xx/100
- **资金流分析**：流入/流出/分化，核心股交叉验证
- **股票池验证**：成功x只/中性x只/失败x只
- **模拟账户台账**：期初/买入/佣金/持仓/现金/总资产/盈亏/收益率/风险暴露
- **成功点/失败点**：3-5条具体列出
- **明日优化**：具体可执行动作

### 语言风格
- 数据驱动，不编造
- 口径差异标注「待补/不强行计算」
- 资料来源与口径说明

## to_docx.py 使用陷阱

**错误用法**：`python3 to_docx.py input.md output.docx`
- 会导致空白文档，因为脚本从stdin读取，不是文件路径

**正确用法**：`cat report.md | python3 to_docx.py "标题" output.docx`
- 通过管道将markdown内容传入stdin
