# Cron报告Patch验证失败修复

**日期**: 2026-05-25
**问题**: 收盘复盘cron job生成报告后尝试patch修改，但找不到匹配的字符串。

## 错误现象

```
hunk '池别汇总' not found — Could not find a match for old_string
```

## 根因

1. Agent生成报告时用了"观察池现状"作为章节名
2. 后续patch尝试用"池别汇总"查找替换
3. 字符串不匹配导致patch失败

## 修复方案

### 1. 明确章节名（在cron prompt中）

```markdown
## 第四步：生成报告
- **🎲 观察池表现**（涨跌榜/异动回顾）——注意章节名必须是"观察池表现"
- **📝 观察池自动管理**（退池/入池/当前池内统计）——注意章节名必须是"观察池自动管理"，**禁止使用"池别汇总"**
```

### 2. 禁止后续patch

```markdown
**一次性写入完整报告，不要后续patch修改**
```

### 3. 同步更新备份job

主job和备份job的prompt必须保持一致的章节名规范。

## 通用规则

- Cron报告生成应一次性写入完整内容，避免后续patch
- 如果必须修改，确保old_string与实际生成的内容完全匹配
- 章节名在prompt中明确指定，不留歧义
