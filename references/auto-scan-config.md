# 观察池自动扫描配置

## 扫描策略（4类）

### 1. ML信号扫描（每日16:00）
- 范围：观察池活跃股票
- 条件：ML评分>60，信号为"买入"
- 入池：C池，验证期1天

### 2. 行业轮动扫描（每日16:30）
- 范围：资金流入前3行业
- 条件：行业龙头，市值前3
- 入池：C池，验证期1天

### 3. 新闻热点扫描（每日17:00）
- 范围：当日新闻热点
- 条件：有股票代码提及，优先关键词命中
- 入池：C池，验证期1天

### 4. 财报扫描（财报季每日）
- 范围：业绩预告/快报
- 条件：净利润增长>30% + 关键词命中
- 入池：C池，验证期1天

## 脚本
pool_auto_scanner.py 支持：
- --ml-only：只扫描ML
- --industry-only：只扫描行业
- --news-only：只扫描新闻
- --financial-only：只扫描财报
- --max N：最大入池数量

| 行业扫描 | 36e009b53ce6 | 16:30 | 每个交易日 |
| 新闻扫描 | 8b4c6c5326f6 | 17:00 | 每个交易日 |
| 验证检查 | e44aa2dd5c8a | 18:00 | 每天 |

## 入池流程
1. 扫描发现候选
2. 过滤（排除已在池中、ST股）
3. 生成逻辑假设
4. 入池（默认C池）
5. 等待评分更新
6. 1天后验证

## Pitfall
1. 新闻缓存可能过期：amadeus_news_scanner.py频繁超时，降级用缓存
2. AKShare限流：批量获取需sleep 0.5秒
3. ML模型可能全部看空：当前市场环境下ML评分普遍偏低
4. 行业扫描依赖AKShare：API不稳定，需备用数据源
