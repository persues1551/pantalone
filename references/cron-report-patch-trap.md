# Cron报告生成陷阱 — 2026-05-25

## 问题现象

收盘复盘cron job报告生成后，agent尝试用patch修改章节名，但找不到匹配字符串：

```
错误: hunk '池别汇总' not found — Could not find a match for old_string
文件: closing_report_20260525.md
```

## 根因分析

1. agent生成报告时使用了"观察池现状"作为章节名
2. 后续尝试patch时用了"池别汇总"（模板中不存在的名称）
3. 字符串不匹配导致patch失败

## 正确做法

### 1. 一次性写入
```python
# 错误：先生成再修改
write_file("report.md", initial_content)
patch("report.md", "旧内容", "新内容")  # ❌ 可能失败

# 正确：一次性写入完整内容
write_file("report.md", complete_content)  # ✅
```

### 2. 章节名必须精确
```markdown
<!-- 模板中的实际章节名 -->
## 📝 观察池自动管理

<!-- 错误的章节名（不存在） -->
## 池别汇总  ❌
```

### 3. Cron Prompt规范
```markdown
## 生成报告
按模板生成报告：
- **🎲 观察池表现**（涨跌榜/异动回顾）——注意章节名必须是"观察池表现"
- **📝 观察池自动管理**（退池/入池/当前池内统计）——禁止使用"池别汇总"
```

## 教训

1. **禁止生成后patch** — 报告必须一次性写入
2. **章节名必须与模板完全一致** — 不要凭记忆猜
3. **Prompt中明确禁止错误章节名** — 防止agent使用不存在的名称
